# [PRD] BizTalk: 업무 말투 변환 서비스

## 1. 프로젝트 개요
**BizTalk**은 사용자가 일상적인 말투로 입력한 업무 내용을 수신 대상(상사, 동료, 고객 등)에 적합한 격식 있는 비즈니스 언어로 자동 변환해주는 AI 기반 서비스입니다. 커뮤니케이션 비용을 줄이고 업무 효율을 높이는 것을 목표로 합니다.

## 2. 프로젝트 목표 및 비전
*   **핵심 가치**: "생각은 자유롭게, 표현은 프로페셔널하게"
*   **주요 목표**:
    *   비즈니스 커뮤니케이션에 어려움을 느끼는 사용자(신입사원 등)의 심리적 부담 완화
    *   메시지 작성 및 교정 시간 단축
    *   상황에 맞는 정확하고 예의 바른 업무 톤앤매너 제공

## 3. 타겟 사용자
*   **신입 사원**: 상사나 외부 고객에게 보내는 메시지 작성이 서툴고 조심스러운 직장인
*   **실무 담당자**: 바쁜 업무 중 정중한 답변을 빠르게 작성해야 하는 기획/운영자
*   **기타**: 비즈니스 커뮤니케이션 능력을 보완하고 싶은 모든 사용자

## 4. 기능적 요구사항 (Functional Requirements)

### 4.1 핵심 기능: 말투 변환 (Tone Conversion)
사용자가 입력한 원문을 선택한 수신 대상에 맞춰 LLM(Large Language Model)이 실시간으로 변환합니다.

| ID | 기능명 | 상세 내용 |
| :--- | :--- | :--- |
| **F-01** | 원문 입력 | 사용자가 변환하고자 하는 내용을 자유롭게 텍스트로 입력 (Textarea) |
| **F-02** | 수신 대상 선택 | 4가지 페르소나 중 하나를 선택 (상사/임원, 타팀 동료, 고객/외부, 팀 내 동료) |
| **F-03** | AI 변환 실행 | 입력값과 페르소나를 기반으로 Upstage Solar-Pro2 모델을 호출하여 결과 생성 |
| **F-04** | 결과 화면 출력 | 변환된 텍스트를 사용자에게 보여주며, 수정 가능한 형태로 제공 |
| **F-05** | 결과 복사 | 변환된 텍스트를 클릭 한 번으로 클립보드에 복사하는 기능 |

### 4.2 수신 대상별 페르소나 정의
| 대상 | 변환 스타일 가이드라인 |
| :--- | :--- |
| **👔 상사 / 임원** | 매우 격식 있고 공손한 경어체 사용, 결론 위주의 간결한 보고 형태 |
| **🤝 타팀 동료** | 정중하면서도 협조적인 어조, 명확한 요청 사항 전달 중심 |
| **🌐 고객 / 외부** | 친절하고 신뢰감을 주는 서비스 어조, 사과나 감사의 표현 포함 |
| **👥 팀 내 동료** | 격식을 차리되 실무 위주의 간결하고 명확한 문체 |

## 5. 비기능적 요구사항 (Non-Functional Requirements)
*   **응답 속도**: 변환 버튼 클릭 후 3~5초 이내에 결과가 출력되어야 함 (LLM 성능 의존)
*   **사용성**: 별도의 가이드 없이도 직관적으로 사용할 수 있는 단일 페이지 UI
*   **보안**: API Key 등의 민감 정보는 환경 변수(`.env`)로 관리하며 클라이언트 측에 노출하지 않음

## 6. 기술 스택 (Tech Stack)
*   **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)
*   **Backend**: Python 3.11+, FastAPI
*   **AI Engine**: Upstage Solar-Pro2 (via LangChain 연동 권장)
*   **Deployment**: Vercel (Frontend), Backend (Local 실행 또는 외부 서버)

## 7. 구현 범위 및 제외 사항 (Scope)

### ✅ 포함 사항 (MVP)
*   단일 페이지 구성의 웹 UI
*   FastAPI 기반의 변환 API 엔드포인트
*   Upstage Solar-Pro2 모델 연동
*   GitHub을 통한 소스 관리 및 Vercel 정적 배포

### ❌ 제외 사항
*   사용자 인증 및 로그인 (회원가입 기능 없음)
*   변환 이력 저장 및 DB 연동
*   고도화된 텍스트 에디터 기능
*   다국어 지원 (한국어 전용)

## 8. 구현 로드맵 (One-Day Plan)
1.  **환경 준비 (30m)**: API 키 발급, 프로젝트 구조 생성, 의존성 설치
2.  **백엔드 구현 (90m)**: FastAPI 서버 구축, LLM 프롬프트 설계 및 연동
3.  **프론트엔드 구현 (60m)**: UI 레이아웃 작성, API 호출 로직 및 결과 출력 구현
4.  **검증 및 배포 (30m)**: 최종 동작 확인 및 GitHub/Vercel 배포
